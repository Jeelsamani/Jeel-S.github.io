Methodology

How It Works

40+ behavioral features are engineered from raw CRM activity logs: days between stages, email response latency percentiles, meeting-to-demo ratio, champion engagement velocity, stakeholder count growth, and 35 more.

XGBoost is trained on historical won/lost deals. SHAP values explain each prediction — so a rep knows not just that a deal is at risk, but why: "champion engagement dropped 40% in the last 7 days."

Scores update daily using a lightweight inference pipeline triggered by CRM activity webhooks — giving every rep a live health check on every open deal.

Model
XGBoost classifier with 40 engineered features. Trained on 2,400 closed deals (balanced). AUC-ROC 0.89 on holdout set.
Explainability
SHAP TreeExplainer produces per-deal feature contributions — showing each rep exactly which behavioral signals are helping or hurting their deal.
Novel Contribution
Using temporal behavioral signal trajectories (not snapshots) as features — treating a deal's engagement health as a time series, not a static record.
